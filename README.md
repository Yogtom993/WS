# TechMarket — завершене вебрішення

## Загальна інформація

**TechMarket** — це навчальний багатосторінковий вебсайт інтернет-магазину техніки. Проєкт створено для демонстрації базових навичок верстки HTML, CSS, роботи з JavaScript, формами, таблицями, адаптивною сіткою, CSS Grid та Bootstrap.

Тематика сайту — продаж ноутбуків, смартфонів та аксесуарів. Вебрішення має зручну структуру, головну сторінку, каталог товарів, сторінку контактів, сторінку реєстрації, а також окремі сторінки для демонстрації CSS Grid і Bootstrap.

---

## Мета проєкту

Метою виконання вебрішення є створення навчального сайту, який демонструє:

- побудову HTML-структури багатосторінкового вебсайту;
- використання CSS для оформлення сторінок;
- застосування адаптивної верстки для різних розмірів екрана;
- створення навігаційного меню;
- роботу з формами введення даних;
- використання таблиць, карток товарів, iframe та медіаблоків;
- підключення JavaScript для інтерактивності;
- демонстрацію CSS Grid;
- демонстрацію Bootstrap-сітки.

---

## Тема вебрішення

Тема сайту — **інтернет-магазин техніки TechMarket**.

Сайт умовно представляє магазин, у якому користувач може переглянути товари, ознайомитися з перевагами магазину, порівняти його з конкурентами, перейти до каталогу, скористатися формою зв’язку або зареєструватися.

Проєкт є навчальним, тому він не містить реальної системи оплати, особистого кабінету, бази даних або серверної частини.

index.html

Головна сторінка сайту. На ній розміщено:

шапку сайту з логотипом і меню;
головний банер із коротким описом магазину;

Ця сторінка є стартовою і відкривається першою.

catalog.html

Сторінка каталогу товарів. Вона містить:

заголовок каталогу;
короткий опис сторінки;

На сторінці працює JavaScript-фільтрація. Користувач може натиснути на категорію, і на екрані залишаться тільки відповідні товари.

contacts.html

Сторінка контактів. На ній реалізовано:

контактну форму;

iframe-карту;
медіаблок із прикладом використання відео.

Після натискання кнопки надсилання JavaScript показує повідомлення про успішну підготовку повідомлення до відправлення.

register.html

Сторінка реєстрації користувача. Вона містить:

поле для логіна;
поле для email;
поле для пароля;
список вибору категорії інтересу;
checkbox для погодження з правилами сайту;
кнопку реєстрації.

Форма не надсилає дані на сервер, а працює як навчальний приклад. Після заповнення та натискання кнопки з’являється повідомлення про успішну реєстрацію.

grid.html

Сторінка демонструє використання CSS Grid.

На сторінці створено макет із такими частинами:

шапка сторінки;
бічне меню;
основний блок;
додатковий блок;
підвал сторінки.

На великих екранах блоки розташовуються у вигляді сітки, а на малих екранах перебудовуються в одну колонку.

bootstrap.html

Сторінка демонструє використання Bootstrap 5.

У ній підключено Bootstrap через CDN і створено блоки з класами:

col-12 — блок займає всю ширину на малих екранах;
col-md-6 — блок займає половину ширини на середніх екранах;
col-lg-4 — блок займає третину ширини на великих екранах.

Ця сторінка показує принцип адаптивної сітки Bootstrap.

картки товарів;

Адаптивність

Сайт адаптований під різні розміри екранів.

У файлі style.css використано media queries:

@media (max-width: 900px) { ... }
@media (max-width: 560px) { ... }

На великих екранах блоки можуть розміщуватися в кілька колонок. На планшетах і смартфонах структура перебудовується в одну або дві колонки, щоб сайт залишався зручним для перегляду.

Наприклад:

блоки переваг на головній сторінці переходять в одну колонку;
каталог товарів з чотирьох колонок переходить у дві, а потім в одну;
навігаційне меню на малому екрані стає вертикальним;
CSS Grid-макет перебудовується в одну колонку.
JavaScript-логіка

У проєкті використано просту JavaScript-логіку.

Фільтрація каталогу

Кнопки фільтрації мають атрибут data-filter, а картки товарів мають атрибут data-category.

При натисканні на кнопку JavaScript перевіряє категорію товару і показує тільки ті картки, які відповідають вибраній категорії.

Категорії:

all — усі товари;
laptop — ноутбуки;
phone — смартфони;
accessory — аксесуари.
Обробка форм

Форми не відправляють дані на сервер. Для навчального прикладу використано event.preventDefault(), щоб сторінка не перезавантажувалася.

Після натискання кнопки користувач бачить повідомлення:

у контактній формі — повідомлення підготовлено до відправлення;
у формі реєстрації — реєстрацію виконано успішно.
